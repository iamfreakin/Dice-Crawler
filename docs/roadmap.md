# Dice Crawler — 로드맵 / 마일스톤

> 작업 우선순위와 목표 분량. 기획 의도는 [design.md](design.md), 진행 체크리스트는 루트 `CLAUDE.md` "현재 상태".

## 전략: "항상 플레이 가능한 완성품"을 유지하며 키운다

대량 콘텐츠(적 12~18, 유물 30~50)를 한 번에 목표하면 지친다.
중간에 **출시 가능한 수직 슬라이스(1막)**를 끼워 완성도부터 올린 뒤, 양을 확장한다.

---

## 작업 순서 (현재 → 출시)

### Phase 1 — 토대: 전투 계산 분리 `[다음]`
- `BattleManager`의 정산을 **순수 계산 `preview(굴린결과, 타겟) → {피해, 방어, 화상, 시너지...}`** 와 **적용**으로 분리.
- 모든 후속 작업(미리보기·면 조각·연출·밸런스)의 기반.

### Phase 2 — 정보 전달(가독성)
- 굴리는 중 **예상 피해/방어 실시간 표시** (Phase 1의 preview 사용)
- 적 의도 **결과 미리 계산**("막고 나면 HP 얼마")
- 상태이상 **툴팁**
- 로그를 안 읽어도 상황 이해 가능하게.

### Phase 3 — 정체성: 면 조각 시스템 ⭐
- 데이터 기반 모디파이어 `ENHANCE / REPLACE / EXPAND` (design.md 참조)
- "주사위 선택 → 면 선택 → 적용" UI (보상/상점/휴식 공유)
- 면 효과를 데이터/전략으로 분리(enum+분기 탈피)
- 특수 면 ~10종 추가

### Phase 4 — 손맛(juice)
- 선행: 전투 UI를 **전체 재생성 → 노드 갱신** 구조로 리팩토링
- 주사위 굴림 애니, 타격/화상/방어 연출, 화면 흔들림, 데미지 숫자 팝업, 적 사망 연출
- 효과음 + BGM

### Phase 5 — 1막 콘텐츠 채우기 (아이콘 아트 병행)
- 아래 "1막 수직 슬라이스" 분량까지 .tres + 아이콘으로 확장
- 이벤트 노드, 2~3 빌드 방향 성립

### Phase 6 — 출시 기능
- 중간 저장 / 이어하기 (Phase 1~ 에서 직렬화 가능한 상태 유지 전제)
- 설정(음량/해상도/전체화면), 일시정지/재시작/종료 확인
- 튜토리얼, 키보드/패드 조작, 엔딩 + 크레딧

## 콘텐츠 목표

### 1막 수직 슬라이스 (먼저 완성할 마일스톤)
- 적 **8종** / 엘리트 **2** / 보스 **1**
- 유물 **~15** / 특수 면 **~10**
- 빌드 방향 **2~3개** 성립
- 첫 클리어 **40~60분**, 완성도(시스템·연출·UI) 높게

### 상업 출시 권장 최소치 (확장 목표)
- 적 **12~18종** / 엘리트 **4+** / 보스 **3**
- 유물 **30~50종** / 특수 면 **15~25종**
- 이벤트 노드 **10+종**
- 빌드 방향 **2~4개**
- 첫 클리어 **2~4시간**

---

## 현재 보유 (데모)
- 주사위 3 (공격/방어/스킬), 적 4 (고블린/박쥐/오크정예/드래곤보스), 유물 4
- 시스템: 에너지 적응형 전투, 개별 리롤, 상태이상, 시너지, 다중 적/타겟, 상점/골드, 분기 맵
- 비주얼: 픽셀 에셋 연결, 폰트, HP바, 패널, 맵 연결선

→ **시스템은 거의 갖춰졌고, "면 조각(정체성) + 정보 전달 + 손맛 + 콘텐츠"가 남은 핵심.**

# 에셋 스펙 (픽셀아트) — Dice Crawler

> **화풍: 픽셀아트.** 원본은 작게 그리고 게임에서 정수배 확대. 텍스처 필터 Nearest(`default_texture_filter=0`).
> **2026-06 크기 개정: 몬스터(적) 제외 전부 2배로 키움** (작게/흐릿해 보이는 문제 해결).

## 현재 아트 상태와 최종 제작 도구

> **현재 `assets/`의 이미지 에셋은 전부 임시본이다.** 게임 시스템과 UI 배치를 검증하기 위한
> 플레이스홀더이며, 출시 전 플레이어·적·주사위·아이콘·배경·UI를 포함해 전체 교체한다.

- 최종 픽셀 에셋과 스프라이트 애니메이션은 **Pixel Composer**를 주 제작 도구로 사용한다.
- 현재는 Pixel Composer 사용 경험이 없으므로, 본격 아트 패스 전에 기본 사용법과
  Godot용 PNG/스프라이트시트 출력 과정을 별도 작업으로 익힌다.
- 제작 범위에는 정적 이미지뿐 아니라 **주사위 굴림·리롤·착지**, 속성/피격 효과,
  적 피격·사망, 맵 노드 발광, 배경 루프 애니메이션도 포함한다.
- 주사위 애니메이션은 **공통 몸체 애니메이션과 결과 면 표시를 분리**한다.
  Pixel Composer에서 몸체 프레임을 만들고 Godot에서 재생 타이밍과 최종 결과 면을 합성한다.
- 원본 프로젝트 권장 경로: `art_source/pixel_composer/`
- 게임에서 사용하는 출력물: 기존 `assets/sprites/`
- 추후 논의할 항목: Pixel Composer 기본 워크플로, 팔레트/외곽선 규칙,
  애니메이션 프레임 수·FPS, 스프라이트시트 레이아웃, Godot 임포트/재생 구조.

## 공통 규칙
- 경로: `assets/sprites/<종류>/<파일명>.png` — **파일명 = 코드 id** (소문자 영문, 정확히 일치)
- 형식: PNG, **투명 배경(알파)**. 배경 이미지만 불투명
- **안티에일리어싱 끄기**, **1px 깔끔한 외곽선**, 제한된 팔레트, 미리 키워 그리지 말 것
- 게임 팔레트: 배경 `#16161a`, 패널 `#1f1f27`, 보라 액센트 `#7f77dd`
- 속성 색: 화염 `#e2503a` / 빙결 `#37c4dd` / 번개 `#efc127` / 저주 `#7a3aa0`
- 등급 색: 일반 `#b4b2a9` / 레어 `#378add` / 에픽 `#7f77dd` / 유니크 `#ef9f27` / 레전더리 `#e24b4a`

## ② 적 — `assets/sprites/enemies/` **(크기 유지)**
구도: 정면/3·4 측면, **발이 캔버스 하단 중앙**, 머리 위 여백. 외곽 1px 라인.

- `goblin.png` (48×48) — 작고 비열한 잡몹. 녹색 피부, 누더기, 녹슨 단검.
- `cave_bat.png` (48×48) — 날개 편 박쥐, 공중에 뜬 포즈. 어두운 보라 + 빨간 눈.
- `orc_elite.png` (64×64) — 덩치 큰 엘리트, 도끼. 외곽 보라 글로우.
- `dragon_boss.png` (96×96) — 보스. 날개·뿔, 가장자리 적색 발광. 캔버스 꽉 차게.

## ⑤ 주사위 면 아이콘 — `assets/sprites/faces/` (**32×32**)
특수 면만. 숫자 면은 폰트 텍스트라 불필요. 속성 색 정확히.
- `fire.png` — 불꽃, 주황빨강(#e2503a)
- `ice.png` — 눈결정, 시안(#37c4dd)
- `lightning.png` — 번개, 노랑(#efc127)
- `curse.png` — 해골, 암보라(#7a3aa0)
- `reroll.png` — 이중 회전화살표, 보라(#7f77dd) *(현재 스킬 면엔 미사용 — 예열로 교체됨)*
- `amplify.png` — 위쪽 화살표/증폭 파동 + `+2`, 노랑(#efc127) `[미수령]`
- `preheat.png` — 다음 굴림 강화(앞쪽 화살표 ▶ + 잔열/불씨), 주황(#e2503a) `[미수령]`
- `transform.png` — 속성 변환(순환 화살표 ⟳ + 속성 결정), 청록/에메랄드 계열 `[미수령]`
  - 참고: 변환 면 3종(🔥/❄️/⚡)은 코드상 모두 `kind=TRANSFORM` → 같은 `transform.png` 사용.
    면에서 목표 속성 구분이 필요하면 속성 색으로 틴트한 3종(`transform_fire/ice/lightning.png`)으로 분리 검토.
- `duplicate.png` — 복제(겹친 사각형/＋1 모티프), 청록/에메랄드 계열 `[미수령]`

## ⑧ 맵 노드 — `assets/sprites/map/`
상태별 색조/연결선/애니메이션은 코드 처리. 코드가 **ring 크기 기준으로 자동 스케일** → 아래 크기대로 덮어쓰면 자동 적용.

### 필수 — 노드 타입 아이콘 (각 **64×64**)
실루엣이 또렷하게, 2~3색, 두꺼운 외곽선.
- `node_battle.png` — 검(전투)
- `node_elite.png` — 해골(정예)
- `node_rest.png` — 모닥불(휴식)
- `node_shop.png` — 상점 간판/주머니(상점)
- `node_boss.png` — 왕관/드래곤 두상(보스)

### 노드 판/테두리
- `node_frame.png` (**80×80**) — 아이콘 뒤 빈 원판. 상태별 색조는 코드가 입힘.
- `node_ring.png` (**96×96**, 가운데 투명) — 현재 노드 발광 테두리.

### 연결선
- `path_dot.png` (**16×16**) — 연결선 점선용.

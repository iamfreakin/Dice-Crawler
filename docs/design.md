# Dice Crawler — 게임 기획 (living doc)

> 이 문서는 **현재 구현 상태 + 설계 방향**을 같이 담는다.
> 각 항목에 `[구현됨]` / `[부분]` / `[예정]` 표시. 코드가 정본이고, 이 문서는 의도를 설명한다.

## 정체성 (한 줄)

> **굴림 결과를 읽고 즉흥 전술을, 면 조각으로 장기 빌드를 짜는 주사위 로그라이크.**

차별점은 **두 시간 축의 결합**:
- **단기(이번 턴)** — 주사위를 하나씩 굴려 나온 눈을 보고 다음 행동을 적응적으로 결정
- **장기(런 전체)** — 주사위 면을 직접 조각(강화/교체/확장)해 덱의 정체성을 빚음

"주사위판 Slay the Spire"에 머물지 않으려면 이 두 축을 항상 전면에 둔다.

플랫폼: PC / itch.io 우선. 엔진: Godot 4.6 + GDScript. 화풍: 픽셀아트.

## 핵심 루프 `[구현됨]`

1. **노드 맵 탐색** — StS식 분기 (전투 / 정예 / 휴식 / 상점 / 보스), 좌표 기반 + 연결선
2. **전투** — 에너지로 주사위를 굴리고, 결과 보고 적응적으로 행동
3. **보상 / 상점** — 새 주사위 / 면 조각 / 유물 / 골드 소비
4. **반복 → 보스** — 빌드 쌓으며 층 클리어

## 전투 시스템

### 턴 흐름 `[구현됨]`
`DRAW → PLAYER_TURN → RESOLVE → ENEMY_TURN → (REWARD)` — `BattleManager` 상태머신.

- **드로우**: 풀(덱)에서 **5장** 핸드로. 매 턴 **에너지 3** 회복(이월 X). 풀 소진 시 버린 더미 셔플.
- **굴림(적응형)**: 핸드의 주사위를 **클릭 = 그 자리에서 즉시 굴림**(에너지 비용 소모). 나온 눈을 보고 다음 주사위를 고른다. 에너지 부족분은 굴림 불가(흐림).
- **개별 리롤**: 굴린 주사위 우상단 **전용 리롤 버튼** → 그 주사위만 재굴림(**리롤 토큰** 1 소모). 본체 클릭은 무반응(오클릭 방지).
- **확정**: 굴린 결과를 한 번에 정산(데미지/방어/효과/시너지), 핸드 전부 버림 → 적 턴.

### 주사위 비용 `[구현됨]`
`DiceData.energy_cost` — 공격/방어 **1**, 스킬 **2**. 에너지 배분이 곧 전술 결정.

### 기본 주사위 3종 `[구현됨]`
| 주사위 | 타입 | 면 구성 | 비용 |
|--------|------|---------|------|
| 공격 | ATTACK | 1~6 숫자 (데미지) | 1 |
| 방어 | DEFENSE | 1~6 숫자 (방어/실드) | 1 |
| 스킬 | SKILL | 🔥3 ❄️3 ⚡3 💀 🔄 2 | 2 |

시작 덱: 각 2개씩(공격2/방어2/스킬2) `StarterDeck`.

### 면 종류(FaceKind) `[구현됨]`
`NUMBER` / `FIRE`(화상) / `ICE`(빙결) / `LIGHTNING`(번개) / `CURSE`(저주, 리롤 토큰+) / `REROLL`(리롤 토큰+).

### 정산 규칙 `[구현됨]`
- 공격/스킬 숫자 면 → 데미지, 방어 숫자 면 → 방어(실드)
- 🔥 면 → 데미지 + **화상** 부여 / ❄️·⚡ → 데미지
- 데미지/효과는 **선택한 타겟 1마리**에게 (단일 타겟 모델)

### 시너지 `[구현됨]`
한 턴에 같은 속성 면 2개 이상:
- 🔥🔥 → 화상 강화 / ❄️❄️ → 적 **약화** / ⚡⚡ → 적 **취약**

### 상태이상 `[구현됨]`
- **화상(burn)**: 적 턴 시작마다 피해 + 1 감소
- **약화(weak)**: 적 공격력 ×0.75 (지속 턴)
- **취약(vulnerable)**: 적이 받는 피해 ×1.5 (지속 턴)

### 적 / 의도 `[구현됨, 공개범위 부분]`
적 등급 NORMAL/ELITE/BOSS. 의도(IntentData): 돌격/저격/폭발/강화/증원 — 머리 위 아이콘+수치.
- `[부분]` 현재 의도는 전부 공개. 기획상 엘리트=일부, 보스=페이즈만 공개는 `[예정]`.
- `[예정]` 증원(SUMMON) 효과 실제 구현(현재 로그만).

### 다중 적 + 타겟 `[구현됨]`
전투당 적 1~여러 마리. 적 클릭으로 공격 대상 지정(타겟 표시). 사망 시 타겟 자동 이동.

## 면 조각 시스템 (핵심 빌드업) `[예정 — 최우선]`

지금은 보상의 "랜덤 숫자 면 +1"만 있고 **핵심이 비어있다.** 설계 방향:

- **데이터 기반 모디파이어**로 구현 (하드코딩 금지):
  - `ENHANCE` — 면 값 상향 (예: 3→5)
  - `REPLACE` — 면 종류 교체 (숫자 → 특수, 또는 특수 → 특수)
  - `EXPAND` — 6면 → 8면, 슬롯 추가
- 보상/상점/휴식/이벤트가 **같은 모디파이어를 공유** → 한 곳에서 "주사위 선택 → 면 선택 → 적용" UI
- **특수 면 콘텐츠(15~25종)**는 이 위에 데이터로 추가
- ⚠️ 확장성 주의: `FaceKind` enum + resolve 분기 방식은 면이 늘면 비대해짐. **면 효과를 데이터/전략으로 분리**(각 면이 자기 효과 적용법을 보유)하는 설계를 면 조각과 함께 도입.

## 기술 아키텍처 `[구현됨]`

- **Autoload**: `GameManager`(런 상태), `SceneRouter`(씬 전환)
- **데이터(Resource + .tres)**: `DiceData`/`FaceData`/`EnemyData`/`IntentData`/`RelicData`
  - 로더 `Content.load_dir/load_one`, 팩토리(`StarterDeck`/`EnemyFactory`/`RelicFactory`)가 .tres 로드
  - 콘텐츠 추가 = `resources/*.tres` 파일 추가 (코드 수정 최소)
- **전투**: `BattleManager`(상태머신) + `EnemyInstance`(런타임 상태)
- **맵**: `MapGenerator`(분기 생성) + `MapNode` + `MapCanvas`(좌표 배치·연결선·노드 아이콘)
- **UI**: 씬은 `.tscn` 노드 구조(에디터 편집), 동적 요소만 코드 생성. 전역 테마 `UITheme`
- 씬: `MainMenu` / `Map` / `Battle` / `Reward` / `Shop`

### 세이브 대비 `[예정]`
풀 세이브 UI는 나중이지만, `GameManager` 런 상태(HP/덱/유물/맵/골드)를 **언제든 dict로 직렬화/복원** 가능하게 유지한다.

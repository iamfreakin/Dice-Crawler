## 정체성 (한 줄)

> **면을 조각하고, 굴리는 순서를 설계하며, 나온 결과에 맞춰 연쇄를 완성하는 주사위 로그라이크.**

세 동사가 우리의 축이다:
- **조각(장기)** — 주사위 면을 직접 강화/교체/확장해 덱 정체성을 빚음
- **순서 설계(단기)** — 주사위를 하나씩 굴리고, 직전 결과를 보고 다음 굴림을 적응적으로 결정. **굴리는 순서 자체가 게임성**
- **연쇄 완성** — 직전 결과를 참조하는 면(촉매/반향/연결 등)으로 콤보를 잇는다

차별점: 다이시 던전은 "굴린 눈 ↔ 장비 매칭"이 핵심이고 주사위는 1~6 도구일 뿐.
우리는 **주사위 면 자체를 조각**하고 **굴림 순서로 연쇄를 설계**한다 — 표면은 닮았으나 축이 다르다. 이 차별점을 항상 전면에 둔다.

플랫폼: PC / itch.io 우선. 엔진: Godot 4.6 + GDScript. 화풍: 픽셀아트.

## 전투 시스템

### 턴 흐름 `[구현됨]`
`DRAW → PLAYER_TURN → RESOLVE → ENEMY_TURN → (REWARD)` — `BattleManager` 상태머신.

- **드로우**: 풀(덱)에서 **5장** 핸드로. 매 턴 **에너지 3** 회복(이월 X). 풀 소진 시 버린 더미 셔플.
- **굴림(적응형)**: 핸드의 주사위를 **클릭 = 그 자리에서 즉시 굴림**(에너지 비용 소모). 나온 눈을 보고 다음 주사위를 고른다. 에너지 부족분은 굴림 불가(흐림).
- **개별 리롤**: 굴린 주사위 우상단 **전용 리롤 버튼** → 그 주사위만 재굴림(**리롤 토큰** 1 소모). 본체 클릭은 무반응(오클릭 방지).
- **확정**: 굴린 결과를 한 번에 정산(데미지/방어/효과/시너지), 핸드 전부 버림 → 적 턴.

### 주사위 비용 `[구현됨]`
`DiceData.energy_cost` — 공격/방어 **1**, 스킬 **2**. 에너지 배분이 곧 전술 결정.

### 기본 주사위 3종 `[구현됨]`
| 주사위 | 타입 | 면 구성 | 비용 |
|--------|------|---------|------|
| 공격 | ATTACK | 1~6 숫자 (데미지) | 1 |
| 방어 | DEFENSE | 1~6 숫자 (방어/실드) | 1 |
| 스킬 | SKILL | 🔥3 ❄️3 ⚡3 💀 🔄 2 | 2 |

시작 덱: 각 2개씩(공격2/방어2/스킬2) `StarterDeck`.

### 면 종류(FaceKind) `[구현됨]`
`NUMBER` / `FIRE`(화상) / `ICE`(빙결) / `LIGHTNING`(번개) / `CURSE`(저주, 리롤 토큰+) / `REROLL`(리롤 토큰+).

### 정산 규칙 `[구현됨]`
- 공격/스킬 숫자 면 → 데미지, 방어 숫자 면 → 방어(실드)
- 🔥 면 → 데미지 + **화상** 부여 / ❄️·⚡ → 데미지
- 데미지/효과는 **선택한 타겟 1마리**에게 (단일 타겟 모델)

### 시너지 `[구현됨]`
한 턴에 같은 속성 면 2개 이상:
- 🔥🔥 → 화상 강화 / ❄️❄️ → 적 **약화** / ⚡⚡ → 적 **취약**

### 상태이상 `[구현됨]`
- **화상(burn)**: 적 턴 시작마다 피해 + 1 감소
- **약화(weak)**: 적 공격력 ×0.75 (지속 턴)
- **취약(vulnerable)**: 적이 받는 피해 ×1.5 (지속 턴)

### 적 / 의도 `[구현됨, 공개범위 부분]`
적 등급 NORMAL/ELITE/BOSS. 의도(IntentData): 돌격/저격/폭발/강화/증원 — 머리 위 아이콘+수치.
- `[부분]` 현재 의도는 전부 공개. 기획상 엘리트=일부, 보스=페이즈만 공개는 `[예정]`.
- `[예정]` 증원(SUMMON) 효과 실제 구현(현재 로그만).

### 다중 적 + 타겟 `[구현됨]`
전투당 적 1~여러 마리. 적 클릭으로 공격 대상 지정(타겟 표시). 사망 시 타겟 자동 이동.

## 면 조각 시스템 (핵심 빌드업) `[예정 — 최우선]`

지금은 보상의 "랜덤 숫자 면 +1"만 있고 **핵심이 비어있다.** 설계 방향:

### 면 조각 모디파이어 (조각 행위) `[예정]`
보상/상점/휴식/이벤트가 **같은 데이터 기반 모디파이어를 공유** → "주사위 선택 → 면 선택 → 적용" UI.
- `ENHANCE` — 면 값 상향 (예: 3→5)
- `REPLACE` — 면 종류 교체 (숫자 → 특수, 특수 → 특수)
- `EXPAND` — 6면 → 8면, 슬롯 추가 *(밸런스 위험 큼 → 후순위)*

### 특수 면: 주사위 조작 (A — 핵심 채택) `[부분]`
단순 데미지 면을 넘어, **굴리는 순서가 의미를 갖게** 하는 조작 면. 적응형 루프의 실제 게임성.
- **증폭** — 직전 결과 +2 *(결과 시점)* `[구현됨]`
- **예열** — 다음에 굴릴 주사위 결과 +2 *(굴림 시점, pending)*
- **복제** — 직전 결과를 복사해 하나 더 *(결과 시점)*
- **보존** — 결과를 다음 턴까지 유지 *(턴 넘김, 1~2개 상한)*
- **변환** — 숫자 결과를 특정 속성으로 변경 *(결과 시점)* → 시너지 완성용
- 예: `공격5 → 복제 → 공격5 둘` / `얼음 → 보존 → 다음 턴 얼음 하나로 시너지 완성`
- ⛔ **분할(6→3+3) 보류**: 결과 수·시너지 횟수·에너지 경제를 동시에 흔들어 밸런스/복잡도 급증.

### 특수 면: 연쇄 (우리만의 축) `[예정]`
직전 결과를 참조해 콤보를 잇는다. **다이시와 가장 확실히 갈라지는 부분.**
- **촉매** — 직전 결과가 속성이면 효과 2배
- **반향** — 직전 특수 면을 다시 발동
- **마무리** — 이번 턴 마지막 굴림이면 강화
- **연결** — 직전 결과와 같은 종류면 에너지 환급

### 조건부 효과 (숫자 빌드 살리기) `[예정]`
숫자 면이 의미를 갖게. **문법은 읽기 쉽게 제한**: 상한 / 짝수·홀수 / 직전 결과 비교.
- `4 이상이면 피해 +3` / `짝수면 방어 +2` / `정확히 6이면 결과 복제` / `직전 결과와 같은 속성이면 강화`

### 효과 적용 시점(timing) — 구현 규약
효과 데이터에 **`timing` 단계**를 둔다(꼬임 방지). 6단계 파이프라인:
- `COST_CALC` — 봉인, 비용 감소 *(비용 계산 상태)*
- `BEFORE_ROLL` — 빙결 적용, 예열 **소비**
- `AFTER_ROLL` — 증폭/복제/변환/촉매/반향/연결, 예열 **생성**
- `ON_CONFIRM` — 데미지/방어/시너지 총합, **마무리**(마지막 굴림 여부는 확정 때만 앎)
- `TURN_START` — 과부하, 보존 복원
- `TURN_END` — 보존 지정, 지속시간 감소
> 효과는 **`trigger`(언제 발동) ≠ `apply`(언제 적용)** 가 분리될 수 있다 — 예열·보존 같은 "지연" 효과는 둘을 따로 둔다.

### 면 효과 실행 모델 (확장성 규약) `[부분 — Phase 3-A]`
`FaceKind` enum은 **행동 로직이 아니라 속성 태그**(시너지 판정용)로만 남긴다. 면은 다중 효과 가능:
```
FaceData
  tags:    [FIRE, ELEMENTAL]        # 시너지/조건 판정용 태그
  effects: [DAMAGE 3, APPLY_BURN 2] # 데이터로 나열된 효과 + 각자 timing
```
`.tres`엔 **데이터만**, 실제 실행은 공용 **`EffectResolver`** 가 담당(분기 로직 한 곳에 집중). 면이 늘어도 코드 분기 대신 데이터만 추가.

`[구현됨]` 기존 숫자/화염/얼음/번개/저주/리롤 면은 `tags + effects(timing)`로 이관 완료.
`[구현됨]` `AFTER_ROLL` 증폭 효과와 순서 replay가 연결됐다. `BEFORE_ROLL` 및 지연 상태는 후속 조작 면에서 확장한다.

### MVP (먼저 만들 최소 세트)
조작 면 **증폭 → 예열 → 변환 → 복제 → 보존** (이 순서로) + 적 디버프 **빙결·봉인·과부하 3종**(정화 수단 먼저).

## 적의 주사위 방해 (B — 긴장감 양념) `[예정]`

적이 **내 주사위 경제를 공격**하게 해 긴장감을 준다. 단, 양념이므로 제한적으로.
**4원칙(불쾌함 방지)**: ① 전투 끝나면 복구 ② 의도로 미리 공개 ③ 어떤 주사위가 영향받을지 예측 가능 ④ 방어/정화 수단 제공.

추천 디버프:
- **빙결** — 다음 굴림의 숫자 상한을 3으로 제한
- **봉인** — 지정 주사위 비용 +1
- **과부하** — 다음 턴 에너지 -1
- **오염** — 한 면을 **전투 중에만** 임시 저주로 교체

> **영구적 저주 면 주입**은 일반 적이 아니라 **엘리트·이벤트 보상 선택**(리스크-리워드)에 어울린다.

### 롤 컨텍스트 (정산 토대) `[부분 — Phase 1/3-B1]`
"굴리는 순서가 게임성"이 되도록 정산은 **순서·맥락**을 가진 이벤트 로그를 replay한다.

**상태 4계층 분리** (한 구조에 다 넣지 말 것):
- **RunState** (`GameManager`) — HP/덱/유물/골드/맵 *(이미 있음)*
- **CombatStatus** — 적, 적이 건 디버프(빙결/봉인/과부하)
- **TurnState** — 에너지, 보존(이월) 결과, 다음 턴 보정
- **RollContext** — 이번 턴 굴림의 **순서·rng_token·확정결과·pending** (RollEntry 이벤트 로그)

**핵심 규약 — 이벤트 로그 + replay (리롤 옵션 2 채택):**
RollContext는 단순 리스트가 아니라 **처음부터 재생 가능한 이벤트 로그**. 항목 구조:
```
RollEntry
  entry_id
  hand_index
  die_id
  rng_token        # 이 굴림에 쓴 고정 난수 — 정규화 [0,1)

ResolvedRoll       # replay마다 새로 생성되는 파생 결과
  entry_id / hand_index / die / face
  value / tags
  generated / copy_depth / source_entry_id
```
- `RollEntry`는 원본 이벤트만 저장하며 선택 면과 계산값을 저장하지 않는다.
- `preview_rolls()`는 `RollEntry`를 건드리지 않고 매번 새 `Array[ResolvedRoll]`을 생성한다.
- **난수(rng_token)와 분포(그 시점 가용 면)를 분리.** 원본 눈을 통째로 굳히지 않고 **토큰**을 굳힌다 → 봉인/오염처럼 분포를 바꾸는 효과가 와도 같은 토큰이 결정론적으로 새 면을 가리킴(유령 면 방지).
- `rng_token`은 **정규화 [0,1)** — 면 선택 `floor(token × 면수)`. EXPAND/분포 변경에도 면 개수 독립.
- 토큰은 **Combat 단위 시드 RNG 스트림**에서 발급 → 리롤 재발급도 재현 가능(시드 런·버그 재현 보너스).

**단일 굴림 서브 파이프라인** (BEFORE_ROLL의 "분포 변경" vs "값 변형"을 순서로 고정):
```
① 분포 효과 (봉인/오염/EXPAND) → 가용 면 확정
② rng_token으로 면 선택        → ResolvedRoll.face
③ 값 변형 (예열 +2, 빙결 상한3) → ResolvedRoll.value
④ 이 면의 AFTER_ROLL 효과(증폭/복제/변환/연쇄)가 이전 항목 참조
```

**replay / 리롤 / 미리보기 / 확정 규칙:**
- **일반 replay**: 모든 `rng_token` 고정, 효과만 처음부터 재계산.
- **과거 리롤**: 해당 항목의 `rng_token`만 새로 발급 → 이후 항목은 기존 토큰으로 재해석 → 연쇄 순서대로 재계산(콤보 재구성, 도박 아님).
- **미리보기**: 동일 replay 실행하되 게임 상태에 **commit 안 함** (= "적용만 안 한 replay").
- **확정**: 동일 결과를 딱 **한 번만 commit**.
- ⚠️ UX: 과거 리롤이 뒤를 조용히 바꾸면 혼란 → 리롤 시 **바뀐 칩 하이라이트/애니**(juice Phase에서).

**무한 콤보 가드:** 생성 결과에 `generated`/`copy_depth`/`source_entry_id` 부여. 생성된 결과는 복제·반향을 **다시 발동 안 함**. 추가로 **턴당 효과 발동 횟수 상한**(안전장치).

이게 면 조작·연쇄·조건부·예상피해의 **공통 토대**다.
